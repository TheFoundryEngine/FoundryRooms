# Team C Wave 0: Foundation & Contracts Implementation Plan

> **Workflow note:** Team C uses an Odin-style foundation/implementation split — implementation sessions write code, foundation session verifies and commits. This file is a cross-team-readable summary; the canonical task prompts live in `.odin/tasks/` of Matt's local foundation workspace and are not in this repo.

**Goal:** Scaffold all four Team C bounded contexts plus the cross-cutting worker app, define every external contract, fixture-pattern mock adapters, and document W0 conventions in a feature ADR. Unblocks Teams A, B, and Bryan/notifications for parallel work.

**Architecture:** Inherits Team A's Wave 0 layout — `modules/<context>/`, `worker/`, `contracts/{api,events,fixtures,mocks}/`. Adds a new `contracts/jobs/` directory for BullMQ payload schemas. Each module follows hexagonal structure (`domain/`, `application/`, `adapters/{inbound,outbound}/`, `contracts/`, `tests/`).

**Tech Stack (inherited from Team A W0):** TypeScript, NestJS, Zod for schema validation, Vitest for testing, npm workspaces, Node 20, `@foundry/contracts` package alias.

**Hard prerequisite:** Team A Wave 0 implementation plan (`docs/superpowers/plans/2026-04-30-wave-0-foundation-contracts.md`) must land on `main` first. Specifically: root `package.json` + npm workspaces, `tsconfig.base.json`, `contracts/` workspace package, `BaseEvent` schema in `contracts/events/identity-access/base.event.ts`.
